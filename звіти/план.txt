Проект - Веб застосунок із  Базою даних мелодій.

МЕТА проєкту (справжня) - захист курсової / в перспективі дипломної.

Мета проєкту (ідеалістична) - створення інструменту виявлення схожих, подібних або ідентичних мелодій для виявлення плагіату, запозичень і впливів або фактів випадкових збігів у мелодіях музичних творів. Актуальність пов'язана із комерційними запитами (питання захисту авторських прав) та науковими (різні музикознавчі дослідження щодо стилістики, запозичень, впливів і т.п.) 

Запланована технічна реалізація: SQL База даних, ASP.NET Razor Pages для створення сторінок проекту та crud-запитів для роботи з БД, сховище для збереження мелодій у форматі midi, бібліотека NAudio для створення інструменту з порівняння та озвучення мелодій.  Azure.Net для розміщення БД.

ЕТАПИ

І. БАЗОВИЙ (термін - 2 тижні)

1. Створення моделі для БД. Моделі (+ - обов'язкові поля)

Melody:   
ID+,  
File (назва файлу, зберігатимуться у wwwroot, string), 
Ttile+ (назва пісні, string) 
Author+ (модель, див. нижче)
Year (дата створення, int)

Author: 
Surname (string)+ 
Name (string)
Country (string)
DateOfBirth (int)
DateOfDeath (int)

Примітка: для народних пісень заздалегідь створити Name "народна", програмно передбачити, що Дати birth/death в цьому випадку недоступні для заповнення користувачем.

2. Створення  сторінок з CRUD запитами для кожної з моделей. Для моделі Melody передбачити кнопку завантаження midi-файлу з комп'ютера користувача. Midiфайли завантажувати в wwwroot/melodies (програмно унеможливити завантаження інших форматів). Кнопки створення та відтворення файлів залишити для наступного етапу.

3. Створення системи автентифікації користувачів. Передбачити, що пошук і додавання  елементів моделей доступний усім користувачам, видалення та редагування - лише адміністраторам.

4. Запуск і тестування моделі. 

5. Спробувати завантажити на Azure.

ІІ. ПЕРCПЕКТИВНИЙ (термін - 3 місяці)

1. Розробити можливість відтворення мелодій безспосередньо в браузері. 
- для цього або знайти js-бібліотеку, або використовувати інструменти NAudio з генеруванням звуку синусоїдою в mp3, збереження його на сервері і потім відтворення через <audio>.

2. Розробити сторінку створення мелодій.  
- Реалізація: додається інтерфейс у вигляді фортепіанної клавіатури, кожна з клавіш викликає POST-запит для додавання ноти у  мелодію. 
- Додається поле з відображенням введених нот у вигляді списку. До кожної ноти - список тривалостей, який викликає POST-запит для встановлення тривалості. За замовченням тривалість четвертна, а також кнопка для вилучення ноти.  
- Додається кнопка для тестового програвання мелодії (див. п.1). 

3. Розробити систему порівняння мелодій. 
- Програмна частина:  C#, алгоритми порівняння колекцій int-значень (всі ноти кодуються як int, що відповідає номеру клавіші), методи порівняння - з урахуванням тривалостей і без, з урахуванням транспозиції і без, повні і часткова схожість. 
- Інтерфейс: як в п.3, крім того поле, в якому після здійснення пошуку відображаються знайдені схожі мелодії (або надпис "жодної подібної мелодії не знайдено").
